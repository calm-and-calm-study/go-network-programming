# UDP 요약

### 개요
- UDP는 IP 상에서 동작하는 비연결형 전송 프로토콜
- 신뢰성 보장, 순서 보장, 흐름 제어 등을 제공하지 않음
- 간단하고 빠르며, 오버헤드가 낮은 전송 방식을 제공

### 특징
- 비연결형 (Connectionless): 데이터 전송 전에 연결을 설정하지 않음
- 신뢰성 없음: 전송 보장, 재전송, 순서 보장 기능 없음
- 오버헤드 낮음: 헤더가 작고 단순함 (8바이트)

### 헤더 구조
필드	크기 (bit)	설명
Source Port	16	송신 측 포트 번호
Destination Port	16	수신 측 포트 번호
Length	16	헤더 + 데이터 전체 길이
Checksum	16	오류 검출용 체크섬

- 체크섬은 선택적이지만 IPv6에서는 필수
- 체크섬은 데이터가 손상되었는지 확인하는데 사용

### 용도
- 데이터 손실을 일부 허용할 수 있는 경우
- 속도가 중요한 실시간 서비스
- 애플리케이션이 자체적으로 오류 처리를 하는 경우

### 추가 기능
- 멀티캐스팅 절차를 통해 하나의 패킷을 복제하지 않고 여러 노드로 전송 가능
- 서브넷 내의 모든 노드에게 패킷을 브로드 캐스팅 할 수 있음

### 파편화 (fragmentation)
- UDP 패킷이 너무 커서 한 번에 전송할 수 없을 때, IP 계층에서 여러 조각(fragment)으로 나누어 전송하는 것.
- MTU보다 클 때 파편화 필요
- MTU보다 큰 패킷이 전송될 때 이더넷 MTU는 보통 1500바이트, IP 헤더(20바이트) + UDP 헤더(8바이트)를 빼면, 실제 UDP 페이로드는 1472바이트 이하여야 함
- ping 으로 MTU 알아내어 파편화 되지 않도록 하는게 필요(ping을 막으면..?)
